# Caspian Admin — Панель управления мессенджером

Веб-приложение на Flutter для администрирования корпоративного мессенджера колледжа.  
Работает в паре с бэкендом [CaspianMessenger.Server](../CaspianMessenger.Server) (ASP.NET Core + PostgreSQL).

## Требования

| Flutter   | ≥ 3.24 |
| Dart      | ≥ 3.5  |
| Браузер   | Chrome / Edge (для Web-режима) |

## Быстрый старт

```bash
# 1. Установить зависимости
flutter pub get

# 2. Запустить в браузере
flutter run -d chrome
```

По умолчанию панель подключается к серверу на `http://localhost:5216`.  
Если адрес другой — поменяй `kBaseUrl` в `lib/core/api/api_client.dart`.

## Структура проекта

```
lib/
├── main.dart                        # точка входа
├── app.dart                         # MaterialApp + тема
├── router.dart                      # маршруты (go_router)
│
├── core/
│   ├── api/api_client.dart          # HTTP-клиент (Dio + JWT-интерцептор)
│   └── storage/token_storage.dart   # хранение токена (SharedPreferences)
│
├── features/
│   ├── auth/                        # авторизация администратора
│   ├── dashboard/                   # главная с графиками и статистикой
│   ├── people/                      # участники (студенты и преподаватели)
│   ├── users/                       # аккаунты мессенджера
│   ├── groups/                      # учебные группы
│   ├── subjects/                    # предметы и назначение преподавателей
│   ├── notifications/               # рассылка push-уведомлений
│   └── import/                      # импорт участников из Excel/CSV
│
└── shared/
    ├── models/                      # общие модели данных
    └── widgets/main_layout.dart     # боковая навигация + обёртка страниц
```

Каждый раздел организован по слоям:

```
features/<name>/
├── data/          # репозиторий (HTTP-запросы, модели)
├── presentation/
│   ├── providers/ # Riverpod-провайдеры (состояние)
│   └── screens/   # UI-экраны
```

## Разделы панели

### 🏠 Главная (Dashboard)
Сводная страница с ключевыми метриками и четырьмя графиками.

- **Карточки:** всего участников, студентов, преподавателей, аккаунтов мессенджера
- **Состав участников** — диаграмма студенты/преподаватели с процентом регистрации
- **Популярные предметы** — топ-6 по количеству назначений
- **Активность** — количество входов в систему по дням (14 дней)
- **Уведомления** — количество рассылок и охват устройств по неделям (8 недель)
- **Прирост участников** — линейный график с нарастающим итогом (30 дней)

Данные обновляются **автоматически каждые 30 секунд**, также доступна ручная кнопка обновления.

---

### 👥 Участники
Справочник всех людей, занесённых в систему (студенты и преподаватели).

- Поиск по ФИО, фильтры по роли и группе
- Добавление, редактирование, удаление записей
- Отображение статуса регистрации (есть ли аккаунт в мессенджере)

---

### 🔑 Пользователи
Аккаунты мессенджера, зарегистрированные участниками.

- Просмотр логина, роли, группы, телефона
- Смена логина и пароля
- Удаление аккаунта

---

### 🗂 Группы
Список учебных групп, сформированных из участников.

- Таблица с количеством студентов в каждой группе
- Раскрываемый список студентов прямо в строке таблицы (inline-раскрытие)
- Для каждого студента отображается статус регистрации в мессенджере
- Удаление группы (удаляет привязку группы у всех её участников)

---

### 📚 Предметы
Управление учебными дисциплинами и назначением преподавателей на группы.

- Добавление и удаление предметов
- Назначение преподавателя на группу по предмету
- Раскрываемая детализация: преподаватель → список групп (inline)
- Снятие назначения

---

### 📣 Уведомления
Массовая рассылка push-уведомлений через Firebase Cloud Messaging.

- Заголовок и текст сообщения
- Вложение файла (изображение)
- Фильтр по аудитории: все / только студенты / только преподаватели / конкретная группа
- История всех отправленных уведомлений с датой и охватом устройств

---

### 📥 Импорт участников
Загрузка списка участников из файла Excel (`.xlsx`) или CSV.

- Перетаскивание файла или выбор через диалог
- Результат: количество добавленных и пропущенных записей
- Пропускаются дубликаты (по совпадению ФИО + группа)

## Первый вход

Перед первым использованием необходимо создать аккаунт администратора.

**Шаг 1.** Убедись, что сервер запущен, и выполни запрос:

```http
POST http://localhost:5216/api/admin/setup
Content-Type: application/json

{
  "login": "admin",
  "password": "твой_пароль"
}
```

Удобнее всего через Swagger: `http://localhost:5216/swagger`

> После создания первого аккаунта endpoint `/api/admin/setup` автоматически блокируется.  
> Последующих администраторов можно добавлять через `POST /api/admin/admins` (требует авторизации).

**Шаг 2.** Открой панель в браузере, введи логин и пароль.

## Конфигурация

### Адрес сервера

`lib/core/api/api_client.dart`:

```dart
const String kBaseUrl = 'http://localhost:5216';
```

Измени на адрес своего сервера (например, `https://api.your-domain.ru`).

### Логотип

Положи файл логотипа по пути `assets/images/logo.png`.  
Он автоматически отобразится в шапке боковой панели и на странице входа.

## Сборка

### Web (продакшен)

```bash
flutter build web --release
```

Результат в `build/web/` — статические файлы, которые можно разместить на любом веб-сервере (nginx, Apache, Firebase Hosting и т.д.).

### Пример конфига nginx

```nginx
server {
    listen 80;
    server_name admin.your-domain.ru;

    root /var/www/admin_panel/web;
    index index.html;

    # SPA: все пути отдаём index.html
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```
